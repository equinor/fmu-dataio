dataio.export.rms package

export_rms_volumetrics(project, grid_name, volume_job_name, global_config='../../fmuconfig/output/global_variables.yml', forcefolder='', subfolder='', name='', tagname='', classification='restricted', workflow='rms volumetric run')[source]

Simplified interface when exporting volume tables (and assosiated data) from RMS.

As the export_volumetrics may have multiple output (storing both tables, maps and 3D grids), the output from this function is always a dictionary. The table is mandatory output, while maps and 3D grid data are optional (not yet implemented).

Parameters:

  • project (Any) – The ‘magic’ project variable in RMS.

  • grid_name (str) – Name of 3D grid model in RMS.

  • volume_job_name (str) – Name of the volume job.

  • global_config (str | Path | dict) – Optional. The global config can either point to the global_variables file, or it can be a dictionary. As default, it assumes a the current standard in FMU: '../../fmuconfig/output/global_variables.yml'

  • forcefolder (str) – Optional. As default, volume tables will be exported to the agreed file structure, and the folder name will be ‘tables’. This can be overriden here, but there will be warnings. For optional assosiated volume maps and grids, the default folder names cannot be changed.

  • subfolder (str) – Name of subfolder for local storage, below the standard folder.

  • name (str) – Optional. Name of export item. Is defaulted to name of grid + ‘_volumes’.

  • tagname (str) – Optional. Defaulted to ‘vol’ for this function. Tagnames are part of file names, and should not be applied as metadata.

  • classification (str) – Optional. Use ‘internal’ or ‘restricted’ (default).

  • workflow (str) – Optional. Information about the work flow; defaulted to ‘rms volumetrics’.

Return type:

dict[str, str]
